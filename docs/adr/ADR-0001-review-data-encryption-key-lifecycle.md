# ADR-0001：复习数据加密与密钥生命周期

## 状态

Proposed

## 决策日期

2026-07-18

## 决策人

待确认

## 背景

`translation-memory-review` 将持久化用户翻译过的词条、语言方向、原始别名、复习进度，以及可再生成的排序、文字和 AI 图片缓存。这些数据可能反映用户阅读或交流内容，不能写入明文 Hive、普通文件、日志或未受保护的缓存。

项目已有 Provider 凭证加密实现，但其主密钥 `.aitrans.provider.key` 与“重置 Provider 凭证”生命周期绑定，而且是 Application Support 下的独立文件。复习历史要求保留至用户主动删除，不能因重置 Provider 凭证而变得不可解密，也不能让普通 Hive 数据与密钥共享同一存储边界。

## 问题

如何在 macOS、iOS、Android 上加密复习历史和全部派生缓存，同时保证：

1. Provider 凭证重置不影响复习历史。
2. 单删/清空能删除历史、进度、别名、排序缓存、文字内容、图片文件及元数据。
3. 密钥缺失、平台安全存储不可用或解密失败时不回退明文，也不隐式生成新密钥覆盖旧密文。
4. 异步排序、文字或图片结果不会在删除后重新创建数据。

## 决策

建议采用以下方案，待负责人确认后方可成为 Accepted：

1. 为复习域建立独立的 `ReviewKeyStore`，在 macOS/iOS 使用 Keychain，在 Android 使用 Keystore 支持的安全存储；不复用 Provider 凭证密钥，也不把复习密钥写入 Hive、图片目录或同一密文仓库。
2. 历史记录、复习状态、原始别名、排序缓存、文字内容、图片字节及元数据使用版本化 AES-256-GCM envelope 加密。AAD 至少绑定数据类型、schema 版本和不含明文词条的稳定记录/内容身份摘要。
3. 历史与派生内容使用独立 repository/box 或加密文件边界；翻译 LRU cache 继续只负责结果复用，不承担复习历史职责。
4. 密钥缺失、读取失败、解密失败或平台安全存储不可用时，复习功能进入明确 `unavailable` 状态；禁止明文降级。仅允许用户显式执行“安全清空并重建”，其顺序为先删除全部密文/派生文件，再删除或重建复习密钥。
5. Provider 凭证保存、删除或重置不得读取、删除或轮换复习密钥。复习清空也不得影响 Provider 凭证。
6. 单删/清空先使记录或仓库 generation 失效，再取消可取消的请求并删除数据；任何晚到异步结果写入前必须校验记录存在、generation、Provider/模型和内容契约仍匹配。
7. 可再生成的派生内容采用独立、有限容量的加密缓存；淘汰只删除派生内容，不删除历史或复习进度。首期容量上限固定为 128 MiB，按最近访问时间淘汰。

## 备选方案

| 方案 | 说明 | 优点 | 缺点 |
|---|---|---|---|
| A. 专用复习密钥 + 平台安全存储（推荐） | 复习域拥有独立密钥和 envelope 生命周期 | 生命周期隔离；不与密文同库；三端安全语义清晰 | 需要平台安全存储 adapter、故障状态和迁移测试 |
| B. 复用 Provider 凭证密钥 | 历史与凭证共享 `.aitrans.provider.key` | 实现最少 | 凭证重置会破坏历史；权限和生命周期耦合；现有密钥文件不属于平台安全存储 |
| C. 新建 Application Support 密钥文件 | 复用 `LocalMasterKeyStore` 模式但使用单独文件 | 与现有代码相似；生命周期可隔离 | 同一用户可同时读取密钥和密文；弱于平台安全存储 |
| D. 明文 Hive 或仅依赖文件系统权限 | 不做应用层 envelope | 开发简单 | 违反已确认加密要求和项目规则，不能接受 |

## 决策原因

该选择同时满足密钥与密文分离、Provider 凭证与复习历史生命周期隔离、三端一致失败语义及显式清空要求。加密格式和密钥生命周期一旦产生用户数据后难以无损切换；如果不记录此上下文，后续实现很容易复用 Provider 密钥或在密钥丢失时错误生成新密钥，因此需要 ADR。

## 高严谨业务系统风险影响

| 维度 | 是否影响 | 影响说明 | 缓解或验证方式 |
|---|---|---|---|
| 领域业务逻辑严谨性 | 是 | 删除和复习状态必须与派生内容一致 | repository 合同和删除闭环测试 |
| 金额与关键数值精度 | 否 | 不涉及金额计算 | 不适用 |
| 交易与数据一致性 | 是 | 跨历史、排序、文字和图片数据删除；晚到响应可能复活数据 | generation、串行/原子 upsert、故障注入测试 |
| 状态流转 | 是 | `ready / unavailable / clearing / reset-required` 等状态需显式 | 状态机测试和用户可见错误 |
| 幂等与并发 | 是 | 重复写入、清空和异步完成存在竞态 | 写入序列化、generation 校验和幂等删除 |
| 权限与审计 | 低度涉及 | 平台安全存储访问受系统权限约束 | 三端 adapter 合同与错误映射 |
| 隐私与适用监管/合规 | 是 | 历史和派生内容包含用户文本及个性化学习状态 | 全量加密、最小化 AI 载荷、彻底删除和无明文降级 |
| 生产变更与回滚 | 是 | schema/key 生命周期变更可能使历史不可读 | 版本化 envelope、迁移 fixture、旧版本读兼容与停止条件 |

## 验证方式

1. 三端安全存储 adapter 覆盖首次创建、读取、缺失、拒绝、损坏和显式重建。
2. AES-GCM envelope 覆盖 nonce、AAD、keyId/schema、篡改、错误密钥和不支持版本。
3. Provider 凭证重置后复习历史仍可读取；复习清空后 Provider 凭证仍可读取。
4. 历史、别名、进度、排序、文字、图片及元数据均不存在明文 fixture 或日志输出。
5. 单删/清空与晚到排序、文字、图片响应并发时，已删数据不会复活。
6. 128 MiB 派生缓存淘汰不删除历史和进度。
