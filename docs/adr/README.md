# Architecture Decision Records

- [KNOWN] Add an ADR only for a difficult-to-reverse or surprising decision with real alternatives.
- [KNOWN] New ADRs start as `Proposed` and require owner confirmation before becoming accepted project context.
- [KNOWN] Use the filename form `ADR-XXXX-short-title.md`.
- [KNOWN] Record context, alternatives, decision, consequences, verification, and follow-up actions.
- [KNOWN] Do not create an ADR merely to document an implementation detail with no meaningful trade-off.
