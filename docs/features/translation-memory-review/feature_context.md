## 2. 需求目标与范围

| 目标 | 说明 | 验收口径 |
|---|---|---|
| [KNOWN] 沉淀可复习翻译记录 | 仅记录用户翻译过的单词和短语，长句不记录 | 同一次翻译响应返回版本化 `word / phrase / sentence / paragraph / unknown` 分类；只记录前两类，并强制排除换行、超过 80 个 Unicode grapheme cluster 或具有多个句子边界的输入 |
| [KNOWN] 推荐可能遗忘的内容 | 确定性间隔重复规则生成到期集合，AI 根据最小候选摘要排序并给出原因 | 用户可看到可解释的到期复习队列；AI 不修改复习状态，失败时按逾期时长、忘记次数和最近复习时间稳定降级 |
| [KNOWN] 提供趣味复习内容 | 复习时展示 AI 记忆插图、词性词义、生活常用语和电影内容 | 图片异步生成、加密缓存并标注“AI 生成”，当前 Provider/模型不支持或失败时显示主题图标；真实台词仅来自应用内批准且有展示权的结构化源，首期无批准源时只显示不带真实片名的“影视化场景对白” |

### 范围外

| 范围项 | 排除原因 | 影响 |
|---|---|---|
| [KNOWN] 长句和段落的复习记录 | 用户明确要求长句不记录 | 仍可正常翻译，但不进入历史和复习推荐 |
| [KNOWN] 账号、跨设备同步、导入、导出和社交排行 | 用户确认首期为三端各自的本地历史与复习 | 换设备或重装后无法从其他设备恢复复习进度；未来如需同步必须另立 Scope |
| 影视剧照、海报、视频播放器、视频 CDN 以及购买、抓取或分发影视视频 | [KNOWN] 用户选择首期方案 1，仅需要短文字台词或非真实影视化场景对白 | 真实影视视频和视觉素材不属于本 Feature 首期范围 |
| 自动执行付费媒体采购或生产操作 | 超出客户端 Scope 和授权边界 | 需另立集成与合规流程 |

### 非目标

- [KNOWN] 不改变基础翻译是否成功的判断。
- [KNOWN] 不将完整翻译历史原文写入日志。
- [KNOWN] AI 排序、文字或图片失败不阻塞基础翻译、历史查看或三档反馈，并进入明确 degraded 状态。
- [KNOWN] 排序、文字和图片调用分别只发送完成该调用所需的词条、语言及必要状态，不发送原始别名、完整历史、长句原文或无关数据。
- [KNOWN] 只有确定性调度器可以修改复习状态；AI 只排序并解释；三档反馈和时间表已确认。

## 4. 核心业务规则

| 规则编号 | 业务域 | 规则说明 | 输入 | 输出 | 边界/例外 | 状态 |
|---|---|---|---|---|---|---|
| TMR-001 | 历史采集 | 同一次翻译响应返回版本化语义分类，只记录 `word`/`phrase`；客户端强制排除换行、超过 80 个 Unicode grapheme cluster 或具有多个句子边界的输入；`sentence`/`paragraph`/`unknown` 不记录 | 源文、纠正后源文、翻译结果、语言对、AI 分类与版本 | `eligible` 或带原因的 `excluded` | 不增加独立分类请求；旧缓存缺少版本化分类时不自动补录，重新翻译后再按新契约判断 | [KNOWN] 用户确认 |
